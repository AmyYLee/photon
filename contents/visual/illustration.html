Illustration

Firefox illustration is approachable, supportive and whimsical.

Principles

Having principles helps to stay true to our visual language and with that to have a more cohesive experience in the Firefox products. Our illustration are approachable, supportive and whimsical.

When to use it

Illustration is a great and powerful tool. For this reason, you must be careful identify the right situation to use it. If the illustration is used only as a decorative element, it may loose its communicative power and distract from the general Firefox experience.

Use illustration to help the user when is frustrated, or lost.

Use illustration to reinforce a feature which is being misued, or neglected.

Use illustration in heavy text component, it can help to explicit the content and encourage to read it.

Illustration can be used to communicate differently with the users, especially when you did to congratulate or celebrate with them.

Where to use it

Illustration must be used with parsimony and only in few context.

Error states

When something goes wrong but not when the user is danger.

On-boarding

When introducing a new feature, or how to use your product.

Long announcements

How to use it

Knowing where to use the right illustration according to the proper context should be done with forethought and attention.

Background colors

Firefox illustration works better with the light grey background #f9f9fa.

Transparency

Firefox illustration is meant to be used always without opacity.

Customization

It's not possible to alter an illustration in any way, or combine it with any other graphics, without the consent of the Content Team.

Dimensions

Illustration always keeps the same ratio. The should take maximum 40% of the available space, minimum 20%.
