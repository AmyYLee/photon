Message bar

Message bars are used to communicate information to the user. The message bars show at the top of the page or section they apply to.

Usage

Message bars contain a concise and scannable text directly related to the action performed or to be performed. They may contain contain a one or more actions, neither of which may be Dismiss or Cancel. They always contain a icon before the message.

Message bars can be used to display different kinds of information such as success messages, updates, warnings and problems.

Types

Generic

Convey general information or actions that arenâ€™t critical. Default banners contain lower priority information and should always be dismissible.

Attention

Use to update users about a change or give them advice.

Success

Inform users when actions are successfully completed

Warnings

Display warning that needs attention or that the merchant needs to take action on. Seeing these banners can be stressful for merchants so be cautious about using them.

Errors

Communicate problems that have to be resolved immediately for the users to complete a task. Seeing these banners can be stressful for the users, so be cautious about using them.

Behaviour

Dismissable

Unless they contain critical information, message bars are dismissible by clicking on the ghost button, placed always on the right corner.

Call to action

Add one or two buttons in the message bar when you want users to take an action after reading the message.
