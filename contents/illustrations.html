Whimsical

When to use it

You should illustration only in certain situation.

  • The users are frustrated, or don’t know what to do next
  • A feature is being bypassed, misused, or underused
  • There’s too much text or the text is not being read and understood
  • We want another way to speak directly to your user
  • We want to congratulate or celebrate with your user

Where to us it

  • Error states (for example, when something goes wrong but not when the user is danger)
  • On-boarding and features (for example, when introducing a new feature, or how to use your product)
  • There’s too much text or the text is not being read and understood< /li>

How to use it

Knowing where to use the right illustration according to the proper context should be done with forethought and attention.
